## 1. 委托构造函数

+ 核心概念

允许一个构造函数（委托者）在其初始化列表中调用同一个类的另一个构造函数（目标构造函数）。

+ 语法：

```c++
class ClassName {
public:
    // 目标构造函数（Target Constructor）
    ClassName(Args... args) { ... }

    // 委托构造函数（Delegating Constructor）
    ClassName(OtherArgs... other) : ClassName(derived_args...) {
         // 额外的初始化逻辑 (可选)
    }
};
```

1. 位置唯一性：委托构造函数的调用必须出现在初始化列表的第一位。不能与其他成员初始化混合。

+ ✅ ```: ClassName(args), member(val)``` (错误！委托必须是唯一的初始化项)

+ ✅ ```: ClassName(args)``` (正确)

+ ❌ : member(val), ClassName(args) (错误)

2. 执行顺序

+ 先执行目标构造函数的主体

+ 目标构造函数返回后，再执行委托构造函数主体中剩余的代码

+ 注意：成员变量的初始化实际上是在目标构造函数中完成的。委托构造函数体中的代码是在所有成员初始化之后执行的

3. 禁止循环委托：A 委托给 B，B 不能委托回 A，否则编译错误（或运行时未定义行为，取决于编译器检测）

4. ```this``` 指针：在目标构造函数执行期间，对象尚未完全构造，因此在目标构造函数中不能使用 ```this``` 指针访问派生类特有的部分（如果是基类的话），或者在委托链中访问尚未初始化的成员

## 2. 继承构造函数

+ 核心概念

允许派生类直接使用基类的构造函数，而不需要手动为每个基类构造函数写一个透传的派生类构造函数。

+ 语法

```c++
class Derived : public Base {
public:
    using Base::Base; // 继承 Base 的所有构造函数
};
```

+ 工作原理

编译器会为 ```Derived``` 生成一系列构造函数，它们的签名与 ```Base``` 的构造函数相同，并在内部调用 ```Base``` 的对应构造函数。

生成的代码逻辑类似于：

```c++
// 编译器隐式生成的代码
Derived(Args... args) : Base(args...) {
    // 初始化 Derived 自己的成员 (使用默认值)
}
```

+ 关键规则与限制

1. 默认成员初始化

继承来的构造函数只会初始化基类部分。派生类新增的成员变量将使用类内默认初始化器 (In-class initializers) 或默认构造函数进行初始化。

+ 如果派生类成员没有默认值且没有默认构造函数，且基类构造函数没有提供该成员的参数，则编译失败

2. 不继承的情况

+ 默认构造函数

如果派生类自己定义了任何构造函数，基类的默认构造函数不会被自动继承（除非显式 using，但如果派生类有成员需要默认构造且无法默认构造，还是会挂）

+ 拷贝/移动构造函数

即使使用了 ```using Base::Base```，派生类的拷贝/移动构造函数不会简单地继承基类的版本。编译器通常会为派生类生成默认的拷贝/移动构造函数（执行成员逐个拷贝），而不是调用基类的拷贝构造。如果需要透传，通常需要手动编写或利用 C++11 的默认规则。

+ 冲突

如果派生类中已经存在与基类签名完全相同的构造函数，则该基类构造函数不会被继承（避免重定义）

3. explicit 属性

如果基类构造函数是 ```explicit``` 的，继承过来的构造函数也是 ```explicit``` 的。
