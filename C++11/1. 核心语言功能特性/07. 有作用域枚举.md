在 C++11 中，有作用域枚举（Scoped Enumerations），通常被称为 ```enum class```（或 ```enum struct```），是为了解决传统 C 风格枚举（```enum```）的诸多缺陷而引入的重要特性。

## 1. 传统枚举（C++98/03）的三大缺陷

在 C++11 之前，使用 ```enum``` 存在以下严重问题：

1. 隐式转换为整数：枚举值可以隐式转换为 int，导致类型不安全

```c++
enum Color { Red, Green, Blue };
int i = Red; // 合法，但破坏了类型封装
Color c = 5; // 合法！可以将任意整数赋给枚举，非常危险
```

2. 作用域污染（Name Pollution）：枚举成员直接泄露到父作用域中，容易导致命名冲突

```c++
enum Color { Red, Green, Blue };
enum TrafficLight { Red, Yellow, Green }; // 错误！Red 和 Green 重复定义
```

3. 底层类型不确定：编译器决定枚举的底层存储类型（通常是 ```int```），无法指定为 char 或 long，这在涉及二进制协议或内存对齐时很麻烦

## 2. C++11 ```enum class``` 的解决方案

C++11 引入了 ```enum class``` 语法，完美解决了上述问题

### 2.1. 强类型作用域（解决命名冲突）

```enum class``` 的成员名称被限制在枚举类的作用域内，必须通过 ```枚举名::成员名``` 访问。

```c++
enum class Color { Red, Green, Blue };
enum class TrafficLight { Red, Yellow, Green }; // 合法！没有冲突

Color c = Color::Red;           // 必须加前缀
TrafficLight t = TrafficLight::Red; // 必须加前缀
```

这避免了全局命名空间的污染，使得不同枚举可以使用相同的成员名。

### 2.2. 禁止隐式转换（解决类型安全）

```enum class``` 不能隐式转换为整数，也不能将整数隐式转换为枚举。必须进行显式强制类型转换。

```c++
enum class Status { Success, Failure };

Status s = Status::Success;
// int i = s;          // 错误！不能隐式转换为 int
int i = static_cast<int>(s); // 正确：必须显式转换

// Status s2 = 0;      // 错误！不能隐式从 int 转换
Status s3 = static_cast<Status>(0); // 正确：必须显式转换
```

### 2.3. 指定底层类型（解决存储大小）

你可以显式指定枚举的底层数据类型，这对于跨平台开发、序列化或与硬件交互非常重要。

```c++
// 指定底层类型为 unsigned char (1 byte)
enum class Permission : unsigned char {
    Read = 1,
    Write = 2,
    Execute = 4
};

static_assert(sizeof(Permission) == 1, "Size should be 1 byte");
```
