在 C++11 中，尾随返回类型（Trailing Return Type） 是一项重要的语法特性，它允许将函数的返回类型声明放在参数列表之后。


这项特性的引入主要为了解决两个核心问题：

1. 复杂类型的可读性：当返回类型非常长或复杂时，将其放在函数名后面会遮挡函数名和参数，导致代码难以阅读。

2. 依赖参数类型的返回类型：在模板编程中，有时返回类型依赖于函数的参数类型。在 C++11 之前，由于参数在返回类型之后声明，编译器无法在解析返回类型时看到参数，导致无法直接表达这种依赖关系。

## 1. 语法格式

```c++
auto 函数名(参数列表) -> 返回类型 {
    // 函数体
}
```

+ ```auto```：在这里作为一个占位符，告诉编译器“返回类型在后面定义”。

+ ```->```：箭头操作符，引导出真正的返回类型。

## 2. 核心应用场景

### 2.1. 简化复杂模板类型的书写

```c++
// 返回类型很长，函数名 findMax 被挤到了右边
typename std::vector<std::pair<int, std::string>>::const_iterator 
findMax(const std::vector<std::pair<int, std::string>>& data);
```

C++11 写法

```c++
// 函数名和参数紧挨着，逻辑清晰
auto findMax(const std::vector<std::pair<int, std::string>>& data) 
    -> std::vector<std::pair<int, std::string>>::const_iterator;
```

### 2.2. 返回类型依赖参数

```c++
template <typename T, typename U>
auto add(T t, U u) -> decltype(t + u) {
    return t + u;
}
```
