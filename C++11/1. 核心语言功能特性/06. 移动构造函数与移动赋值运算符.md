在 C++11 中，移动语义（Move Semantics） 是革命性的特性，它通过移动构造函数（Move Constructor）和移动赋值运算符（Move Assignment Operator），极大地提升了资源管理（如内存、文件句柄、网络连接）的效率。

它的核心思想是：“与其深拷贝一份资源，不如直接‘偷’走对方的资源。”

## 1. 核心概念： 左值 VS 右值

要理解移动语义，必须先区分左值（lvalue）和右值（rvalue）：

+ 左值：有名字、有持久地址的对象（如 ```int a = 10;``` 中的 ```a```）。你可以取它的地址 ```&a```。

+ 右值：临时对象、字面量、即将销毁的对象（如 ```10```, ```a + b```, ```std::move(a)```）。它们通常没有名字，生命周期仅限于当前表达式。

移动语义的规则：

+ 拷贝：用于左值。因为左值后续还会被使用，所以必须复制一份，保证原对象不变

+ 移动：用于右值。因为右值是临时对象，马上就要销毁了，我们没必要复制它的资源，直接把它的资源指针“抢”过来，然后把原对象的指针置空即可

## 2. 移动构造函数

+ 定义：

一个特殊的构造函数，接受一个右值引用（```T&&```）作为参数，将源对象的资源“转移”到新对象中，并将源对象置于有效但未指定的状态（通常是置空）。

+ 语法：

```c++
ClassName(ClassName&& other) noexcept;
```

+ ```ClassName&&```：右值引用，只能绑定到临时对象或显式转换为右值的对象。

+ ```noexcept```：非常重要。标准库容器（如 ```std::vector```）在扩容时，如果元素的移动构造函数标记为 ```noexcept```，会使用移动而不是拷贝，以保证异常安全并提升性能

## 3. 移动赋值运算符

+ 定义

当把一个右值对象赋值给一个已存在的左值对象时调用。它需要先释放当前对象持有的旧资源，然后接管源对象的资源。

+ 语法

```c++
ClassName& operator=(ClassName&& other) noexcept;
```

## 4. 为什么需要 ```std::move```

编译器默认将命名变量视为左值。即使你有一个临时对象想要移动，如果你给它起了个名字，它就变成了左值，调用拷贝构造。

```std::move``` 的作用：

它不进行任何移动操作，它只是一个类型转换工具（强制类型转换），将左值转换为右值引用（T&&），从而触发移动构造函数或移动赋值运算符。

## 5. 规则之五

在 C++11 之前，我们有“规则之三”（析构、拷贝构造、拷贝赋值）。C++11 引入了移动语义后，扩展为规则之五：

如果你的类需要自定义以下任何一个函数，通常意味着你需要自定义全部五个：

1. 析构函数 (```~T()```)

2. 拷贝构造函数 (```T(const T&)```)

3. 拷贝赋值运算符 (```T& operator=(const T&)```)

4. 移动构造函数 (```T(T&&)```)

5. 移动赋值运算符 (```T& operator=(T&&)```)
