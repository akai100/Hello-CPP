## 1. 预置函数（```= default```）

+ 定义：

显式告诉编译器：“请为我生成这个函数的默认实现”。

+ 背景：

在 C++11 之前，如果你手动定义了任何一个特殊成员函数（如拷贝构造函数），编译器就不再自动生成其他默认函数（如默认构造函数或移动构造函数）。这有时会导致不必要的样板代码。

= default 允许你在手动声明其他函数的同时，依然请求编译器生成默认版本。

+ 语法：

```c++
class MyClass {
public:
    // 显式请求编译器生成默认构造函数
    MyClass() = default; 

    // 显式请求编译器生成默认析构函数
    ~MyClass() = default;

    // 显式请求编译器生成默认拷贝构造
    MyClass(const MyClass&) = default;
    
    // 显式请求编译器生成默认移动构造 (C++11 新特性)
    MyClass(MyClass&&) = default;
};
```

## 2. 弃置函数（```= delete```）

+ 定义：

显式告诉编译器：“禁止生成或使用这个函数”。任何尝试调用该函数的代码都会导致编译错误。

+ 背景：

在 C++11 之前，如果想禁止拷贝，通常会将拷贝构造函数和赋值运算符声明为 private 且不提供实现。但这有两个缺点：

1. 如果在类内部或友元中调用，链接时会报错（而不是编译时）

2. 语法意图不够直观
