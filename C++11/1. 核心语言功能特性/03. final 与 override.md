在 C++11 中，final 和 override 是两个用于控制继承体系的关键字。它们不是类型的一部分，而是编译器指令，旨在让代码意图更清晰，并防止常见的继承错误。


## 1. ```override```：显示表明“我要重写”

+ 定义：

放在虚函数声明的末尾，表示该函数必须重写基类中的某个虚函数。

+ 作用：

1. 编译期检查：如果基类中没有匹配的虚函数（签名不匹配、基类函数不是虚函数、或者拼写错误），编译器会直接报错。

2. 提高可读性：阅读代码的人一眼就能看出这个函数是为了重写基类行为，而不是定义一个新的同名函数

+ 语法：

```c++
struct Base {
    virtual void foo(int x);
    virtual void bar();
};

struct Derived : Base {
    // 正确：成功重写 Base::foo
    void foo(int x) override; 

    // 错误！编译报错：Base 中没有 void bar(float) 这个虚函数
    // 可能是你想写 void bar()，但不小心写成了 float 参数
    void bar(float f) override; 
};
```

## 2. ```final```：显示表明“到此为止”

+ 定义：

用于禁止进一步的继承或重写。它可以修饰类，也可以修饰虚函数。

### 2.1. 修饰类（```class FinalClass final```）

+ 作用：该类不能被其他类继承。任何尝试继承它的操作都会导致编译错误。

+ 场景：
  + 工具类/单例类：设计上就不允许被扩展。
  + 性能优化：编译器知道该类不会有派生类，因此可以进行去虚拟化（de-virtualization）优化，直接将虚函数调用转换为普通函数调用，提升效率。
  + 安全性：防止用户通过继承破坏类的不变量（Invariant）。

+ 示例：

```c++
struct Base final {
    virtual void func();
};

// 错误！编译报错：不能继承 final 类
struct Derived : Base { 
};
```

### 2.2. 修饰虚函数（```void func() final```）

+ 作用：该虚函数在当前的类中是最终版本，派生类不能再重写它。但类本身仍然可以被继承，只是不能重写这个特定函数。

+ 场景：
  + 核心逻辑保护：基类提供了某种算法框架（如模板方法模式），其中某个步骤的实现是固定的，不允许子类修改。
  + + 接口契约：确保某些关键行为在所有派生类中保持一致。
