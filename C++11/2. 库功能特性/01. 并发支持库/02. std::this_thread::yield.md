
+ 定义

它提示操作系统调度器：“当前线程愿意放弃剩余的 CPU 时间片，让其他就绪状态的线程（尤其是同优先级的）先运行。”

## 1. 核心机制

+ 非阻塞 VS 阻塞

```yield()``` 不会让线程进入睡眠状态（Sleeping/Blocked）。线程依然处于 Runnable (就绪) 状态。

+ 调度器行为

  + 调用 ```yield()``` 后，当前线程会被移回到就绪队列的末尾（或根据调度策略重新排序）
 
  + 调度器会立即从就绪队列中选择另一个线程运行
 
  + 如果没有其他就绪线程（例如单核 CPU 且只有一个线程在跑，或者其他线程都在等 I/O/锁），调度器可能会立即再次调度当前线程。此时 ```yield()``` 的效果几乎为零，甚至因为上下文切换的开销而变慢

+ 优先级敏感

  它主要让出 CPU 给同等或更高优先级的线程。如果有低优先级线程在跑，高优先级线程调用 ```yield``` 通常不会让低优先级线程抢占（取决于具体 OS 实现）

## 4. 底层实现差异

+ Linux （```pthread_yield``` / ```sched_yield```）

  + 将当前进程放入就绪队列末尾
 
  + 如果有同优先级的其他进程，则切换；否则立即切回自己

+ Widows(```SwitchToThread```)

  + 类似，但行为更复杂，受限于处理器组和优先级提升机制
 
  + 注意：C++ 标准库在 Windows 上通常映射到 SwitchToThread 或类似的 API
