在 C++17 之前，定义多层嵌套的命名空间需要多次重复 namespace 关键字和大括号 {}，导致代码向右缩进严重，不仅难以阅读，也难以维护。

## 1. 语法对比

C++17 之前：

```c++
namespace A {
    namespace B {
        namespace C {
            namespace D {
                void func() {
                    // ...
                }
            }
        }
    }
}
```

缺点：

+ 代码冗长：大量的 namespace 关键字和大括号；

+ 缩进灾难：代码主体被推到了屏幕右侧，降低了可读性

+ 维护困难：如果想重命名中间某一层，需要修改多处大括号的匹配

C++17:

你可以使用 ```::``` 操作符将多层命名空间合并在一行声明中：

```c++
namespace A::B::C::D {
    void func() {
        // ...
    }
}
```

优点：
+ 简洁清晰：一眼就能看出完整的层级结构

+ 减少缩进：函数体可以紧贴左侧，阅读更舒适

+ 易于重构：修改路径只需改一行
