
C++17 临时量实质化 (Temporary Materialization) 是 C++17 标准中对值类别（Value Categories）模型的一次重要重构。它引入了一个新的概念阶段，
用于解释纯右值（prvalue）如何转换为实际存在的对象（xvalue），以便程序可以获取其地址或绑定到引用。

## 1. 核心概念

在 C++17 之前，表达式 ```T()``` 的结果就是一个临时的 ```T``` 对象（右值）

在 C++17 中，表达式 ```T()``` 的结果是一个 纯右值 (prvalue)。

+ Prvalue：表示一个初始化结果的值，但它不指代任何内存中的对象。它没有地址，不能绑定到 const T&（除非经过实质化），也不能直接调用成员函数（除非该函数是静态的或通过特殊规则）

+ Xvalue (eXpiring value)：指代一个具有身份（identity，即有地址）的对象，该对象的生命期即将结束（通常是可以被移动的对象）


临时量实质化 (Temporary Materialization) 是将 prvalue 转换为 xvalue 的过程：

1. 编译器根据 prvalue 的描述，在内存中创建一个临时对象

2. 这个新生成的临时对象成为一个 xvalue

3. 程序现在可以获取它的地址，或将其绑定到引用


## 2. 何时发生实质化

prvalue 不会 自动变成临时对象。只有在以下情况发生时，才会触发临时量实质化：

### 2.1. 绑定到引用

当 prvalue 需要绑定到一个引用（无论是 const T& 还是 T&&）时：

```c++
struct S { int x; };
S makeS() { return {1}; } // 返回 prvalue

const S& ref = makeS(); 
// 过程：makeS() 返回 prvalue -> 触发实质化 -> 创建临时 S 对象 (xvalue) -> 绑定到 ref
```

### 2.2. 取地址操作

当对 prvalue 应用 & 运算符时：

```c++
auto ptr = &makeS(); 
// 过程：makeS() 返回 prvalue -> 触发实质化 -> 创建临时 S 对象 -> 取地址
```

### 2.3. 访问非静态成员

```c++
int val = makeS().x; 
// 过程：makeS() 返回 prvalue -> 触发实质化 -> 创建临时 S 对象 -> 访问 .x
```

### 2.4. 作为函数实参传递（且形参是引用）

```c++
void func(const S& s);
func(makeS()); // 触发实质化
```

## 3. 与强制消除的关系

场景：

```c++
S s = makeS();
```

+ C++ 14 逻辑

1. ```makeS()``` 返回一个临时对象（prvalue 立即具现为临时对象）

2. 该临时对象被移动（或拷贝）到变量 ```s``` 中；

3. 临时对象析构(即使编译器优化掉了步骤 2 和 3，语义上它们必须存在，所以 S 必须有移动构造函数)；

+ C++ 17 逻辑

1. ```makeS()``` 返回一个 prvalue（仅仅是一个初始化指令，无临时对象生成）;

2. 变量 ```s``` 直接使用这个 prvalue 进行初始化

3. 没有发生临时量实质化，因为不需要绑定引用，也不需要取地址。编译器直接在 ```s``` 的内存位置上构造对象

4. 结果：零开销，且不需要 ```S``` 拥有移动构造函数;

结论：由于 prvalue 不再隐式生成临时对象，只有在必要时才实质化，这为“强制复制消除”扫清了语义障碍。在 S s = makeS(); 中，因为没有触发实质化的条件，所以永远不会生成中间临时对象。
