C++17 结构化绑定 (Structured Bindings) 是 C++17 引入的一项非常实用的特性，它允许你将复合类型（如 std::pair, std::tuple, 数组，或自定义结构体/类）中的成员直接绑定到一组独立的变量名上。

## 1. 基本语法

```c++
auto [var1, var2, ..., varN] = expression;
```

+ ```expression``` 必须是一个返回**数组**、**std::pair**、**std::tuple** 或具有公共非静态数据成员（或提供 get<N> 接口）的类的表达式

+ ```var1``` 到 ```varN``` 是新声明的变量名，它们分别绑定到表达式结果对应的元素上

+ 变量的数量必须与元素的数量完全匹配，否则编译报错


## 2. 使用场景

### 2.1 解包 ```std::pair``` 和 ```std::map``` 迭代器

```c++
std::map<std::string, int> scores = {{"Alice", 90}, {"Bob", 85}};
    
// key 绑定到 first, value 绑定到 second
for (const auto& [name, score] : scores) {
    std::cout << name << ": " << score << "\n";
}
```

### 2.2 解包```std::tuple```

```c++
std::tuple<int, std::string, double> getStudentInfo() {
    return {101, "Charlie", 3.8};
}

auto [id, name, gpa] = getStudentInfo();
    
std::cout << "ID: " << id << ", Name: " << name << ", GPA: " << gpa << "\n";
```

### 2.3 解包数组

```c++
int arr[] = {10, 20, 30};
auto [x, y, z] = arr;

x = 100; // 修改 x 不会改变原数组 arr，因为默认是按值拷贝
```

### 2.4 解包自定义结构体

只要结构体的成员是公有的 (public) 且非静态，就可以直接使用结构化绑定。

```c++
struct Point {
    int x;
    int y;
    int z;
};

Point p{1, 2, 3};
auto [px, py, pz] = p;

// 如果是引用绑定，可以修改原对象
auto& [rx, ry, rz] = p;
rx = 10; // p.x 现在变成了 10
```

如果成员是 ```private``` 的，则需要提供特定的 ```get<N>()``` 友元函数或模板特化（类似 ```std::tuple``` 的行为），但这比较复杂，通常建议将用于绑定的数据结构成员设为 ```public``（如在 DTO 模式中）

## 3. 引用与值语义

结构化绑定支持值拷贝、左值引用和右值引用，行为与普通 ```auto``` 声明一致：

### 3.1 值拷贝 (默认)

```c++
auto [a, b] = obj;
```

创建副本，修改 ```a```, ```b``` 不影响原对象。

### 3.2 左值引用

```c++
auto& [a, b] = obj;
```

绑定到原对象的成员，修改 ```a```, ```b``` 会 改变原对象。

要求 ```obj``` 是左值

### 3.3 常量引用

```c++
const auto& [a, b] = obj;
```

绑定到原对象，但不能修改。推荐用于遍历容器以避免拷贝

### 3.4 右值引用

```c++
auto&& [a, b] = getTemporaryObj();
```

用于绑定临时对象，延长生命周期
