
 C++17 开始，在特定场景下，标准强制要求编译器省略拷贝/移动构造函数的调用。如果编译器无法执行这种消除，代码将被视为格式错误（Ill-formed）。
 这也意味着在这些场景下，即使类删除了拷贝/移动构造函数，代码依然可以合法编译。

## 2. 适用场景

### 2.1 返回临时对象

当函数返回一个与返回类型相同的纯右值（prvalue，即临时对象）时：

```c++
struct NoCopy {
    NoCopy() = default;
    // 显式删除拷贝和移动构造函数
    NoCopy(const NoCopy&) = delete;
    NoCopy(NoCopy&&) = delete;
};

// C++17: 合法。编译器必须在 ret 的位置直接构造 NoCopy 对象。
// C++14: 非法。因为需要调用移动构造函数，但它被删除了。
NoCopy createObject() {
    return NoCopy(); 
}

int main() {
    NoCopy obj = createObject(); // 这里也没有拷贝/移动发生
}
```
