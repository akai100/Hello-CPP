```std::optional``` 是 C++17 引入的一个极具价值的类模板，位于 <optional> 头文件中。

它的核心目的是：以一种类型安全、语义清晰的方式表示“一个值可能存在，也可能不存在”的状态。

它主要用来替代以下两种传统的、容易出错的模式：

1. 返回特殊值（Magic Numbers）：例如返回 ```-1```、```nullptr``` 或 ```NaN``` 来表示错误或缺失。这要求调用者必须知道这些特殊值的含义，且如果合法值恰好是 ```-1``` 就会出错。


## 1. 核心概念

```std::optional<T>``` 对象内部包含两个状态：

+ 有值 (Engaged)：包含一个类型为 ```T``` 的有效对象

+ 无值 (Disengaged)：不包含任何对象（类似于空指针，但它是值语义的）

关键特性：

+ 栈分配：optional 对象本身通常分配在栈上（除非 T 本身很大），避免了 new/delete 的开销和内存碎片

+ 显式检查：在使用值之前，编译器强制（或强烈建议）你检查是否有值，避免了空指针解引用崩溃

+ 语义清晰：函数返回 std::optional<T> 明确告诉调用者：“这个函数可能失败或没有结果”，无需查阅文档猜测返回值含义。

## 2. 基本用法示例

### 2.1 查找操作

```c++
std::optional<std::string> find_name(int id, const std::vector<std::pair<int, std::string>>& db) {
    for (const auto& [curr_id, name] : db) {
        if (curr_id == id) {
            return name; // 隐式构造 optional<string>
        }
    }
    return std::nullopt; // 明确表示“无值”
}

 std::vector<std::pair<int, std::string>> database = {
    {1, "Alice"}, {2, "Bob"}, {3, "Charlie"}
};

// 1. 获取结果
auto result = find_name(2, database);

// 2. 检查是否有值 (推荐方式)
if (result.has_value()) {
    std::cout << "Found: " << result.value() << "\n";
} else {
    std::cout << "Not found.\n";
}

 // 3. 更简洁的布尔转换写法
if (result) { 
    std::cout << "Found (short): " << *result << "\n"; // 使用 * 解引用
}

// 4. 提供默认值 (C++17)
std::string name = result.value_or("Unknown User");
std::cout << "Name to display: " << name << "\n";
```

## 3. 常用 API

| 方法/操作 | 描述 | 示例 |
| :--- | :--- | :--- |
| 构造 | 创建有空值或无值的 optional | `std::optional<int> o1 = 10;`<br>`std::optional<int> o2 = std::nullopt;` |
| has_value() | 检查是否包含值 | `if (o.has_value()) ...` |
| operator bool | 隐式转换为 bool (常用) | `if (o) ...` |
| value() | 获取值，若无值则抛 `std::bad_optional_access` | `int x = o.value();` |
| operator*** | 解引用获取值引用，不检查是否有值（若为空则未定义行为） | `int x = *o;` |
| operator-> | 访问成员（当 T 是类时） | `o->member_func();` |
| value_or(default) | 若有值返回值，否则返回提供的默认值 | `int x = o.value_or(0);` |
| reset() | 销毁内部值，变为无值状态 | `o.reset();` |
| emplace(args...) | 就地构造内部值（避免拷贝） | `o.emplace(1, 2, 3);` |
