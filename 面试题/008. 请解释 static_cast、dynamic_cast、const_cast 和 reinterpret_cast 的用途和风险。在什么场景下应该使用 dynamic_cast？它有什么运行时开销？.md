## C++ 四种类型转换详解

| 转换 | 用途 | 安全性 | 典型场景 |
|------|------|--------|--------|
| `static_cast` | 编译期的“合理”类型转换 | ⚠️ 中等（依赖程序员正确使用） | 基本类型转换、有继承关系的指针/引用转换（非多态）、void* 转回原类型 |
| `dynamic_cast` | 运行时安全地在多态类型间转换 | ✅ 高（失败可检测） | 向下转型（base → derived），需 RTTI |
| `const_cast` | 添加或移除 `const` / `volatile` | ⚠️ 危险（可能破坏 const 正确性） | 与 C API 交互（如传入 const 数据但函数参数非 const） |
| `reinterpret_cast` | 低级位模式重解释（几乎无类型检查） | ❌ 极危险 | 硬件编程、序列化、底层系统开发 |

## ```static_cast```

+ 用途：用于“相关类型”之间的转换

+ 例子

```c++
double d = 3.14;
int i = static_cast<int>(d); // 浮点转整型

Base* b = new Derived();
Derived* d = static_cast<Derived*>(b); // 向下转型（⚠️ 不安全！若 b 实际不是 Derived，行为未定义）
```

+ 风险：不检查运行时类型，向下转型可能出错。

## ```dynamic_cast ```

+ 用途：安全地进行多态类型的向下转型或侧向转型

+ 要求：基类必须有虚函数（即启用 RTTI）

+ 行为：

  + 指针版本：失败返回 ```nullptr```；
 
  + 引用版本：失败抛出 ```std::bad_cast```

```c++
Base* b = getSomeObject();
if (Derived* d = dynamic_cast<Derived*>(b)) {
    d->derivedMethod(); // 安全调用
}
```

+ 运行时开销：需要查询虚函数表（vtable）中的类型信息（RTTI），比 static_cast 慢；

+ 何时使用：当你不确定对象的实际类型，又需要安全访问派生类接口时。

## ```const_cast```

+ 唯一合法用途：移除 const（通常是为了调用旧 C API）。


## ```reinterpret_cast```

+ 本质：直接将位模式重新解释为另一种类型。

+ 风险：极易引发未定义行为（如违反 strict aliasing 规则）；

+ 原则：除非写系统级代码，否则永远不要用

## 总结对比表

| Cast | 编译期/运行时 | 安全性 | 是否需要 RTTI | 典型用途 |
|------|---------------|--------|----------------|--------|
| `static_cast` | 编译期 | 中 | 否 | 基本类型转换、向上转型 |
| `dynamic_cast` | 运行时 | 高 | ✅ 是 | 安全向下转型 |
| `const_cast` | 编译期 | 低 | 否 | 移除 const（谨慎！） |
| `reinterpret_cast` | 编译期 | 极低 | 否 | 位级操作（极少用） |


## 面试总结话术

“我优先使用 C++ 风格的 cast 而非 C 风格 (Type)，因为它们意图明确、可搜索、安全性可控。
dynamic_cast 用于多态类型的安全向下转型，但有 RTTI 开销；
const_cast 仅用于与遗留 C 接口交互；
reinterpret_cast 几乎不用，除非在底层系统编程中。”
