+ 必须将基类析构函数声明为 virtual 的情况是：

  + 当通过基类指针（或引用）删除派生类对象时。

+ 如果不这样做：

  + 只会调用基类的析构函数；
 
  + 派生类的析构函数不会被调用；
 
  + 导致派生类中分配的资源（如内存、文件句柄等）无法释放 → 资源泄漏；
 
  + 行为是未定义的（UB）（C++ 标准规定：通过基类指针删除派生类对象，而基类析构函数非虚，是 UB）。

```c++
class Base { public: ~Base() {} };          // 非虚析构
class Derived : public Base { 
    std::vector<int> data; 
public: 
    ~Derived() { /* 清理 */ } 
};

Base* p = new Derived();
delete p; // ❌ UB！~Derived() 不会被调用，data 内存泄漏
```
