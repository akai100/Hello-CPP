## ```explicit``` 的作用

```explicit``` 用于禁止编译器进行隐式类型转换（implicit conversion），强制用户显式调用构造函数。

它通常用于单参数构造函数（或 C++11 起的多参数但可被单一实参列表初始化的构造函数）。

## 为什么需要 ```explicit```？——隐式转换的陷阱

```c++
class MyString {
public:
    MyString(int size) { // 隐式构造函数！
        data_.resize(size);
    }
private:
    std::vector<char> data_;
};

void print(const MyString& s) {
    // ...
}

int main() {
    print(10); // 😱 编译通过！但逻辑错误：把整数 10 当成字符串长度！
}
```

+ 程序员本意是传一个字符串，却误传了 int；

+ 编译器自动调用 MyString(10) 构造临时对象；

+ bug 难以发现，运行时行为诡异。

✅ 使用 explicit 修复：

```c++
class MyString {
public:
    explicit MyString(int size) { // 👈 加 explicit
        data_.resize(size);
    }
};

int main() {
    // print(10); // ❌ 编译错误！不能隐式转换 int → MyString
    print(MyString(10)); // ✅ 必须显式构造
}
```

✅ 好处：

+ 防止意外类型转换；

+ 提高代码可读性和安全性；

+让接口意图更明确。

##  面试总结话术

explicit 用于禁止构造函数的隐式调用，防止意外的类型转换。我通常对所有单参数构造函数加上 explicit，除非该类型确实需要像内置类型一样参与自动转换（比如数值类）。这能显著减少隐蔽 bug，提高代码健壮性。
