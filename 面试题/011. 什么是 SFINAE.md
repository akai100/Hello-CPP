SFINAE（Substitution Failure Is Not An Error）: 中文：“替换失败不是错误”

+ 当编译器尝试实例化一个函数模板时，如果在推导模板参数的过程中发生类型替换失败（比如某个类型不支持 operator+），不会立即报错；

+ 而是将该模板从候选重载集中移除；

+ 只要还有其他合法的重载，程序就正常编译。

用途：根据类型特性有条件地启用/禁用模板。

## 2. 用 SFINAE 实现 ```print_if_integral```

```c++
#include <type_traits>
#include <iostream>

// 启用条件：T 是整数类型
template<typename T>
typename std::enable_if<std::is_integral_v<T>>::type
print_if_integral(T value) {
    std::cout << "Integral: " << value << "\n";
}

// 可选：为非整数提供 fallback（非必须）
template<typename T>
typename std::enable_if<!std::is_integral_v<T>>::type
print_if_integral(T value) {
    std::cout << "Not integral, skipped.\n";
}
```

## 3. C++20 Concepts 如何简化？

C++20 引入 Concepts，让约束更直观、错误信息更友好。

```c++
#include <concepts>
#include <iostream>

// 定义 concept
template<typename T>
concept Integral = std::is_integral_v<T>;

// 使用 requires 子句
void print_if_integral(Integral auto value) {
    std::cout << "Integral: " << value << "\n";
}

// 或显式模板形式
template<Integral T>
void print_if_integral(T value) {
    std::cout << "Integral: " << value << "\n";
}
```
