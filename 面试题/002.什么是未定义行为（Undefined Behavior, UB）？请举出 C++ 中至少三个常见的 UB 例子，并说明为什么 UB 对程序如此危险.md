未定义行为（UB） 是指 C++ 标准没有规定程序在某种情况下应如何执行的行为。一旦发生 UB，程序可能：

+ 崩溃（如段错误）；

+ 输出错误结果；

+ 表现“正常”但隐藏严重 bug；

+ 被编译器优化掉（因为标准允许编译器假设 UB 永远不会发生）。

## 2. 三个常见 UB 示例：

1. 访问数组越界

2. 解引用空指针或野指针

3. 有符号整数溢出

其他常见 UB:

+ 修改字符串字面量（```char* s = "hello"; s[0] = 'H';```）

+ 在构造函数中调用虚函数（可能调用未初始化子类的函数）

+ 数据竞争（多线程同时读写同一非原子变量且无同步）


## 3. 为什么 UB 如此危险？

1. 程序行为不可靠：同一份代码在不同平台/编译器/优化级别下表现不同；

2. 编译器可任意优化：现代编译器会假设 UB 不会发生，从而删除“看似无用”的代码。例如：

```c++
if (x + 1 < x) { /* 溢出检查 */ }
```

3. 安全漏洞根源：缓冲区溢出等 UB 是黑客攻击（如栈溢出）的主要入口。
