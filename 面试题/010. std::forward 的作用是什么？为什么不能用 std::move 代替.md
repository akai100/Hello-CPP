## ```std::forward``` 的作用：

在模板函数中，将通用引用参数“原样”转发给其他函数，保留其左值/右值属性。

## 为什么不能用 ```std::move```？

+ ```std::move(x)``` 总是将 x 转为右值；
  
+ 如果 x 原本是左值（如变量），move 会错误地将其变为右值，导致资源被“偷走”，后续使用出错

```c++
template<typename T>
void wrapper_bad(T&& arg) {
    some_func(std::move(arg)); // ❌ 总是 move！即使 arg 是左值
}

template<typename T>
void wrapper_good(T&& arg) {
    some_func(std::forward<T>(arg)); // ✅ 保留原始值类别
}
```

## 面试总结话术

```
“通用引用是模板中形如 T&& 的参数，能根据实参自动推导为左值或右值引用。
std::forward 利用模板参数类型信息，配合引用折叠，实现‘按原样转发’。
完美转发广泛用于工厂函数和容器的就地构造（如 emplace_back），避免不必要的拷贝。”
```
