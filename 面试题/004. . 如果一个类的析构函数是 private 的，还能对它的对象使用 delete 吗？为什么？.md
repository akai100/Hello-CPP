不能（在类外部）直接使用 ```delete```。

+ ```delete p``` 会隐式调用析构函数；

+ 如果析构函数是 ```private```，且 ```delete``` 表达式出现在类外部，则编译器会报错：“无法访问 private 成员”；

+ 这是一种设计模式：用于强制对象只能在堆上创建，并由类内部控制销毁（例如通过 ```destroy()``` 成员函数）

```c++
class OnlyHeap {
private:
    ~OnlyHeap() = default; // private 析构

public:
    static void destroy(OnlyHeap* p) {
        delete p; // ✅ 类内部可以 delete（有访问权限）
    }
};

int main() {
    OnlyHeap* p = new OnlyHeap();
    // delete p; // ❌ 编译错误！析构函数不可访问
    OnlyHeap::destroy(p); // ✅ 正确方式
}
```

💡 面试加分点：

“将析构函数设为 private 是一种控制对象生命周期的设计技巧，常用于单例或资源管理类。”
