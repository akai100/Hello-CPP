## 1. 什么是虚假唤醒（Spurious Wakeup）？

虚假唤醒 是指：线程在没有被 ```notify_one()``` 或 ```notify_all()``` 显式唤醒的情况下，从 ```wait()``` 中返回。

这听起来很奇怪，但它是操作系统和硬件层面允许的行为，原因包括：

+ 某些 CPU 架构（如 ARM）的底层原语可能产生“假信号”；

+ 操作系统为了性能或中断处理，可能临时唤醒等待线程；

+ POSIX 标准明确允许 ```pthread_cond_wait``` 出现虚假唤醒。


##  2. 为什么必须用 while 而不是 if？

❌ 错误写法（用 if）：

```c++
std::mutex mtx;
std::condition_variable cv;
bool ready = false;

// 消费者线程
{
    std::unique_lock<std::mutex> lock(mtx);
    if (!ready) {               // ⚠️ 只检查一次
        cv.wait(lock);          // 可能虚假唤醒！
    }
    // 此时 ready 可能仍为 false！
    do_work(); // ❌ 危险：条件未满足就执行
}
```

+ 如果发生虚假唤醒，线程会跳出 wait()，但 ready 仍是 false；

+ 由于用了 if，不再检查条件，直接执行后续逻辑 → 严重 bug！

✅ 正确写法（用 while）：

```c++
{
    std::unique_lock<std::mutex> lock(mtx);
    while (!ready) {            // ✅ 循环检查
        cv.wait(lock);
    }
    do_work(); // 安全：只有 ready==true 才继续
}
```

+ 即使虚假唤醒，也会重新检查条件；

+ 如果条件不满足，再次进入 wait()；

+ 只有当 ready == true 时才退出循环。

✅ 更简洁的写法（C++11 推荐）

std::condition_variable::wait 支持带谓词（predicate）的重载，内部自动用 while：

```c++
{
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, []{ return ready; }); // ✅ 等价于 while(!ready) wait();
    do_work();
}
```

## 3. 面试总结

```
“虚假唤醒是指条件变量在没有被通知的情况下意外返回，这是操作系统允许的行为。
因此，我们必须用 while 循环（或带谓词的 wait）反复检查条件，确保只有在真正满足时才继续执行。
这是编写正确多线程代码的基本原则之一。”
```
