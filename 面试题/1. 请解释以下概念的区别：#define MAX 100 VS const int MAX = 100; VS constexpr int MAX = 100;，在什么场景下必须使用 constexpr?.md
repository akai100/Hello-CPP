## ```#define MAX 100```

本质：预处理器指令，在预处理阶段进行纯文本替换；

缺点：

  + 无类型检查：```#define PI 3.14``` 和 ```#define PI "hello"``` 都合法，但语义错误；

  + 作用域问题：宏是全局的，容易命名冲突；

  + 调试困难：编译器看到的是替换后的代码，调试器无法识别 MAX；

  +  副作用风险：如 #define SQUARE(x) x*x，调用 SQUARE(i++) 会导致 i 自增两次

## ```const int MAX = 100;```

本质：一个运行时常量（尽管值在编译期可知）；

优点：

  + 有类型（int），支持类型检查；

  + 遵循作用域规则（可在函数、类、命名空间内定义）；

  + 调试器可识别；

局限：

  + 不能用于需要编译期常量的场景，例如：

```c++
const int n = 10;
int arr[n];        // C++ 中非法！（C 允许，但 C++ 要求编译期常量）
std::array<int, n> a; // 错误！n 不是 constexpr
switch (x) {
    case n: ...    // 错误！case 标签必须是编译期常量
}
```

## ```constexpr int MAX = 100;```

本质：编译期常量表达式（C++11 引入）；

要求：

  + 初始化表达式必须是常量表达式（只含字面量、其他 constexpr 变量/函数等）；

  + 变量本身隐含 ```const```；

优势：

  + 可用于所有需要编译期常量的上下文（数组大小、模板参数、```case``` 标签等）；

  + 有类型、有作用域、可调试；

  + 可用于优化（编译器可将其直接内联为字面量）

##  何时必须使用 ```constexpr```

| 场景 | 示例 |
|------|------|
| 数组大小（C++） | `int a[100];` → 若用变量，必须是 `constexpr` |
| 模板非类型参数 | `template<int N> class Buffer {}; Buffer<10> b;` |
| `case` 标签 | `switch(x) { case 10: ... }` |
| `std::array` 大小 | `std::array<int, N>` |
| 对齐要求 | `alignas(16) char buf[32];` |
| 编译期计算 | `constexpr int factorial(int n) { ... }` |
