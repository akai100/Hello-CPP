在 C++14 中，函数返回类型推导 (Return Type Deduction) 是另一项极具实用价值的特性。

它允许编译器根据 ```return``` 语句自动推断函数的返回类型，从而省去了繁琐的尾随返回类型语法（Trailing Return Type）。

## 1. 核心变化：从 ```auto f() -> T``` 到 ```auto f()```

C++11 的局限

在 C++11 中，auto 只能用于局部变量。如果用于函数返回值，必须配合尾随返回类型语法，因为编译器在解析函数体之前就需要知道返回类型（以便进行参数检查和符号表构建）。

```c++
// C++11: 必须显式指定返回类型，即使它很明显
template <typename T, typename U>
auto add(T t, U u) -> decltype(t + u) {
    return t + u;
}
```

缺点：代码冗余，尤其是当表达式很复杂时，```decltype(...)``` 需要重复写一遍表达式。

C++14 的突破

C++14 允许直接使用 ```auto``` 作为返回类型，编译器会分析函数体内的所有 ```return``` 语句来推导类型。

```c++
// C++14: 简洁清晰，无需重复表达式
template <typename T, typename U>
auto add(T t, U u) {
    return t + u; // 编译器自动推导为 decltype(t + u)
}
```

## 2. 推导规则 (面试重点)

编译器如何确定返回类型？遵循以下逻辑：

1. 单一返回类型：

+ 如果函数所有 return 语句返回的类型一致（或可隐式转换为同一类型），则推导为该类型。

+ 类似于模板参数推导规则。

2. 多个返回语句类型不一致：

+ 如果不同的 return 语句返回了不同且无法统一的类型，编译报错。

```c++
auto func(bool b) {
    if (b) return 10;      // int
    else return 3.14;      // double
    // ✅ 合法：int 和 double 可以统一推导为 double (公共类型)
}

auto func2(bool b) {
    if (b) return 10;      // int
    else return std::string("hi"); // string
    // ❌ 错误：int 和 std::string 没有公共类型，编译失败
}
```

3. 递归函数限制：

+ 关键点：在函数体内调用自身时，由于返回类型尚未推导完成，不能直接使用纯 auto 推导。

+ 解决方案：必须使用 C++11 的尾随返回类型，或者在 C++14 中使用占位符但需前置声明（较麻烦，通常推荐尾随返回类型）。

```c++
// ❌ 错误：递归调用时返回类型未知
auto factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1); // 编译器不知道 factorial 返回什么
}

// ✅ 正确：使用尾随返回类型辅助推导
auto factorial(int n) -> int {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```
