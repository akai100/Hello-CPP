在 C++14 中，聚合类 (Aggregate Classes) 的定义被显著放宽，允许成员变量拥有默认成员初始化器 (Default Member Initializers)。

这是一个非常实用但容易被误解的特性。在 C++11 中，一旦你给结构体成员写了 ```= value```，它就不再是“聚合类”，从而失去了聚合初始化 (Aggregate Initialization) 的能力（即不能使用 {} 列表初始化）。C++14 修复了这个矛盾。

## 1. 核心变化：C++ 11 vs C++ 14

### 1.1. C++ 11 的限制

在 C++ 11 中，聚合类的定义非常严格：

1. 没有用户声明的构造函数

2. 没有私有或受保护的非静态数据成员

3. 没有基类

4. 没有虚函数

5. 关键限制：不能有默认成员初始化器（即不能写 ```int x = 0;```）

如果你在 C++11 中写了默认值，编译器会认为该类不是聚合类，因此不能使用花括号 ```{}```进行初始化，必须显式定义构造函数。

```c++
// C++11 代码
struct Config {
    int id = 0;          // ⚠️ 有了这个，Config 就不是聚合类了
    double rate = 1.0;   // ⚠️ 有了这个，也不是聚合类
};

int main() {
    // ❌ 编译错误：Config 不是聚合类，不能使用列表初始化
    Config c1{10, 2.5}; 
    
    // ✅ 必须手动写构造函数才能这样用，或者只能逐个赋值
    // 或者只能使用默认构造：Config c2; (id=0, rate=1.0)
}
```

### 1.2. C++14 的放松

C++14 修改了聚合类的定义：允许成员拥有默认成员初始化器。

只要满足其他条件（无用户构造、无虚函数等），即使成员有 ```= value```，它依然是聚合类，依然可以使用 {} 初始化。

```c++
// C++14 代码
struct Config {
    int id = 0;          // ✅ 允许默认初始化
    double rate = 1.0;   // ✅ 允许默认初始化
    std::string name;    // 无默认值
};

int main() {
    // 场景 1: 提供所有值 (覆盖默认值)
    Config c1{10, 2.5, "ServerA"}; 
    // 结果: id=10, rate=2.5, name="ServerA"

    // 场景 2: 只提供部分值 (前面的覆盖，后面的使用默认值)
    Config c2{20}; 
    // 结果: id=20, rate=1.0 (默认), name="" (默认构造)

    // 场景 3: 不提供任何值 (全部使用默认值)
    Config c3{}; 
    // 结果: id=0, rate=1.0, name=""
    
    // 场景 4: 跳过中间项 (C++20 指定初始化才支持更好，C++14 只能按顺序)
    // Config c4{.rate = 3.0}; // ❌ C++14 不支持指定初始化，必须按顺序
    // 在 C++14 中，如果想只改 rate，必须写出 id:
    Config c5{0, 3.0, "DefaultName"}; 
}
```

## 2. 初始化优先级规则

当默认成员初始化器和聚合初始化同时存在时，规则非常简单：

+ 显式提供的值 > 默认成员初始化器

+ 如果初始化列表中提供了值，则使用该值

+ 如果初始化列表中省略了该成员（且后面没有更多成员被初始化），则使用类内默认值

```c++
struct Point {
    int x = 10;
    int y = 20;
    int z = 30;
};

Point p1{1, 2, 3};    // x=1, y=2, z=3 (全部覆盖)
Point p2{100};        // x=100, y=20(默认), z=30(默认)
Point p3{};           // x=10, y=20, z=30 (全部默认)
```

## 3. 面试核心价值与应用场景

### 3.1. 为什么这个特性重要？

1. POD/Aggregate 的优势保留：

   + 聚合类可以进行零开销初始化（直接内存填充，无需调用构造函数）
  
   + 支持列表初始化，语法简洁
  
   + 在 C 语言交互、序列化、GPU 计算 (CUDA/OpenCL) 中，聚合类布局更可控
  
   + C++14 让你在享受这些底层优势的同时，还能拥有“默认值”的安全网

2. 减少样板代码 (Boilerplate)

   + 不再需要为了提供默认值而手写默认构造函数
  
   + 不再需要为了支持 {} 初始化而移除默认值并改用 memset 或手动赋值
  
3. 安全性提升

   + 防止未初始化变量。如果调用者忘记传递某个参数，成员会自动回退到安全的默认值，而不是未定义的垃圾值

### 3.2. 典型应用场景

+ 配置结构体：大部分配置项有合理的默认值，用户只需覆盖少数项。

+ 图形/数学库：```Vector3```, ```Color```, ```Rect``` 等，通常有默认构造（如全 0 或全 1），但也需要方便地通过 {x, y, z} 创建。

+ 测试桩 (Mock Objects)：快速构建带有默认状态的对象
