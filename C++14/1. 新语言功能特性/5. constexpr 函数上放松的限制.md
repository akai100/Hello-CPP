## 1. 核心对比

| 特性 | C++11 限制 (极其严格) | C++14 放松 (接近普通函数) |
| :--- | :--- | :--- |
| 函数体语句数 | 只能有一条 `return` 语句 (除了 using 声明和 typedef) | 允许任意多条语句 |
| 局部变量 | 禁止定义局部变量 | 允许定义局部变量 (只要类型是字面量类型) |
| 控制流 | 禁止循环 (`for`, `while`)、条件分支 (`if`, `switch`) | 允许完整的控制流 (`if`, `switch`, `for`, `while`, `break`, `continue`) |
| 变量修改 | 无变量可改 | 允许修改局部变量的值 |
| 空函数 | 不允许 (必须有 return) | 允许空函数 (`void` 返回类型) |
| 静态/线程存储 | 禁止 | 依然禁止 (不能有 `static` 或 `thread_local` 变量) |


## 2. 代码实例对比

### 2.1. 场景：计算阶乘

❌ C++11 写法：被迫使用递归三元运算符

由于只能有一条 return 语句，无法使用 if 或循环，代码可读性极差。

```c++
// C++11: 必须写成这种难以阅读的递归形式
constexpr int factorial(int n) {
    return (n <= 1) ? 1 : (n * factorial(n - 1));
}
```

如果逻辑稍微复杂一点（比如需要中间变量），C++11 几乎无法实现，必须退回到模板元编程 (TMP)，那是另一套复杂的语法体系。

✅ C++14 写法：像写普通函数一样自然

可以使用 ```if```、局部变量 ```res``` 和 ```for``` 循环。

```c++
// C++14: 逻辑清晰，支持循环和局部变量
constexpr int factorial(int n) {
    int res = 1;              // ✅ 允许定义局部变量
    for (int i = 2; i <= n; ++i) { // ✅ 允许 for 循环
        res *= i;             // ✅ 允许修改变量
    }
    return res;               // ✅ 允许多条语句，最后 return
}

// 验证：编译期计算
static_assert(factorial(5) == 120, "Compile time check failed");
```

## 3. 面试深度追问与回答策略

### Q1: C++14 放松 constexpr 限制的目的是什么？

+ 回答要点：

1. 降低门槛：C++11 的 constexpr 限制太多，导致稍微复杂的逻辑就必须使用模板元编程 (TMP)。TMP 语法晦涩难懂，编译错误信息冗长，开发效率低。

2. 统一模型：C++14 让开发者可以用普通的 C++ 语法（循环、变量）来编写编译期代码，实现了“编译期逻辑”和“运行期逻辑”的语法统一。

3. 性能与安全：鼓励将更多计算移至编译期，减少运行时开销，同时保持类型安全。

### Q2: C++14 的 constexpr 函数可以在运行时调用吗？

+ 回答要点：可以。

  + constexpr 关键字的含义是“如果参数是常量表达式，则必须在编译期求值；否则，它就像一个普通的 inline 函数在运行时执行”。
  + 这使得同一份代码既能用于编译期常量（如数组大小、模板参数），也能用于运行时变量，无需重写。

### Q3: C++14 中还有哪些东西是 constexpr 函数里依然禁止的？

+ 回答要点（展示对边界的理解）：

1. 静态变量：不能定义 static 或 thread_local 变量（因为它们有持久状态，不符合纯函数式特征）。

2. 动态内存分配：不能调用 new / delete / malloc / free（编译期没有堆内存概念，虽然 C++20 进一步放宽了部分分配器限制，但 C++14 严禁）。

3. 非字面量类型：局部变量的类型必须是 LiteralType（通常指 trivial 类型或 constexpr 构造/析构的类）。

4. try-catch：不能包含异常处理块（C++14 时期，C++20 有所放宽但仍有限制）。

5. Goto：不能使用 goto 语句。

### Q4: C++14 对 constexpr 构造函数有什么改进？

+ 回答要点：
  + C++11 要求构造函数体必须为空（只能用初始化列表）。
  + C++14 允许构造函数体内包含语句（只要满足上述函数体的放松规则），使得初始化逻辑更灵活。
