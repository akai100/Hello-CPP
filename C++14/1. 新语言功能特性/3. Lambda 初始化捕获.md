在 C++14 中，Lambda 表达式的捕获列表（Capture Clause）得到了显著增强，允许初始化捕获 (Init-capture)，也被称为广义捕获 (Generalized Capture)。

这一特性允许你在捕获变量时直接对其进行初始化，而不仅仅是复制外部作用域的现有变量。这意味着你可以：

1. 移动语义捕获：捕获只能移动不能复制的对象（如 ```std::unique_ptr```）。

2. 重命名捕获：给捕获的变量起一个 Lambda 内部使用的别名。

3. 任意表达式初始化：捕获的值可以是任意表达式的计算结果，而不必是外部已存在的变量


## 1. 语法格式

```c++
[变量名 = 表达式](参数) { 函数体 };
```

+ ```变量名```: Lambda 内部使用的变量名

+ ```表达式```：用于初始化该变量的任意 C++ 表达式（在 Lambda 定义处求值）

+ 可以混合使用普通捕获（```[x]```）和初始化捕获（```[y = x + 1]```）

## 2. 核心应用场景

### 2.1. 移动语义捕获 —— 最重要的用途

在 C++11 中，Lambda 默认按值捕获时会调用拷贝构造函数。如果对象不可拷贝（如 ```std::unique_ptr```, ```std::mutex```, ```std::fstream```），就无法按值捕获。C++14 通过初始化捕获完美解决了这个问题。

场景：将一个独占所有权的资源移入 Lambda，供异步任务或回调使用。

```c++
 auto ptr = std::make_unique<int>(42);

// ❌ C++11 无法这样做，因为 unique_ptr 不可拷贝
// auto lambda = [ptr]() { ... }; // 编译错误

// ✅ C++14: 使用 std::move 将所有权移入 Lambda
// 内部变量名也叫 ptr，外部 ptr 变为 nullptr
auto lambda = [ptr = std::move(ptr)]() {
    std::cout << "Value inside lambda: " << *ptr << std::endl;
};

// 此时外部的 ptr 已经为空
if (!ptr) {
    std::cout << "External ptr is now empty.\n";
}

lambda(); // 输出: Value inside lambda: 42
```

### 2.2 重命名捕获

有时外部变量名很长，或者与 Lambda 内部逻辑名称冲突，你可以给它起个新名字。

```c++
int very_long_variable_name_that_is_annoying = 100;

// 内部使用简短的名字 'x'
auto lambda = [x = very_long_variable_name_that_is_annoying]() {
    return x * 2;
};

std::cout << lambda() << std::endl; // 输出 200
```

### 2.3. 表达式初始化

捕获的值不必是外部现有的变量，可以是任何表达式的计算结果。这在需要“快照”当前状态非常有用。

```c++

std::vector<int> data = {1, 2, 3};
int factor = 10;

// 捕获 data 的大小乘以 factor 的结果，而不是 data 本身
// 即使外部 data 后来被修改，lambda 里的 size_snapshot 也不会变
auto lambda = [size_snapshot = data.size() * factor]() {
    return size_snapshot;
};

data.push_back(4); // 修改外部数据，大小变为 4

std::cout << "Snapshot value: " << lambda() << std::endl; 
// 输出 30 (3 * 10)，不受外部 data 变化的影响

```

## 3. 底层原理

闭包类的成员初始化

```c++11
class Closure {
    std::unique_ptr<int> ptr; // 错误！unique_ptr 没有拷贝构造函数
public:
    Closure(std::unique_ptr<int> p) : ptr(p) {} // 试图拷贝，失败
};
```

c++14

```c++
// [ptr = std::move(p)]()
class Closure {
    std::unique_ptr<int> ptr; 
public:
    // 构造函数接收右值引用并移动
    Closure(std::unique_ptr<int> p) : ptr(std::move(p)) {} 
};
```
